# 🧪 learn-cypress

A dedicated repository for mastering **Cypress**, focusing on modern end-to-end (E2E) testing practices, component testing, and automation workflows.

## 🚀 Purpose

This project documents my journey learning Cypress. It serves as a practical sandbox where I implement test suites to validate UI components, API endpoints, and user flows on real-world applications.

### Tech Stack
* **Framework:** [Cypress](https://www.cypress.io/)
* **Languages:** JavaScript / TypeScript
* **Reporting:** Mocha / Cypress Dashboard

---

## 📚 Learning Roadmap & Milestone Checklists

- [x] **Cypress Architecture & Setup:** Understanding the runner and configuration.
- [x] **Locators & Assertions:** Mastering `cy.get()`, `cy.contains()`, and `.should()`.
- [ ] **Advanced Interactions:** Handling file uploads, drag-and-drop, and iframes.
- [ ] **API Testing:** Using `cy.request()` for integration testing.
- [ ] **Custom Commands:** Writing reusable helpers in `cypress/support/commands.js`.
- [ ] **CI/CD Integration:** Running tests headlessly via GitHub Actions.
